# 快速诊断实验

这些实验用于判断当前 Stage 1 OPT-to-SAR 生成效果差的原因。不要先继续扩大全量训练；先验证小样本能否过拟合，以及单数据集是否比混合多数据集稳定。

## 0. 一键并行诊断

推荐先直接跑一键脚本。它会：

1. 生成 32 对 overfit manifest；
2. 在两张 GPU 上同时跑 `debug_overfit32_base` 和 `debug_overfit32_paired`；
3. 自动生成 `SUMMARY.md`、`summary.json`，并复制最新 panel 到 `latest_panels/`。

```bash
cd /inspire/hdd/global_user/liuxiaotong-253108540242/yanggang/lihao/lh/or/SAR-Generation/sar-sd

FULL_MANIFEST=/inspire/hdd/global_user/liuxiaotong-253108540242/yanggang/lihao/lh/or/SAR-Generation/dataset/stage1_prepared/manifest.jsonl \
GPU_BASE=0 \
GPU_PAIRED=1 \
OVERFIT_EPOCHS=200 \
bash tools/run_quick_diagnostics.sh
```

如果只有一张 GPU：

```bash
GPU_BASE=0 GPU_PAIRED=0 RUN_PARALLEL=0 bash tools/run_quick_diagnostics.sh
```

如果还想继续跑单数据集 2k：

```bash
RUN_SINGLE_DOMAIN=1 SINGLE_DOMAIN_DATASET=SAR2Opt bash tools/run_quick_diagnostics.sh
```

结果位置会在终端最后打印，例如：

```text
runs/quick_diagnostics_20260702_153000/SUMMARY.md
runs/quick_diagnostics_20260702_153000/latest_panels/
runs/quick_diagnostics_20260702_153000/logs/
```

可以打包后发回：

```bash
tar -czf quick_diagnostics_result.tar.gz runs/quick_diagnostics_YYYYMMDD_HHMMSS
```

## 1. 准备小样本 manifest

先设置全量 manifest 路径：

```bash
FULL_MANIFEST=/inspire/hdd/global_user/liuxiaotong-253108540242/yanggang/lihao/lh/or/SAR-Generation/dataset/stage1_prepared/manifest.jsonl
```

查看各数据集数量：

```bash
python - <<'PY'
import json
from collections import Counter

path = "/inspire/hdd/global_user/liuxiaotong-253108540242/yanggang/lihao/lh/or/SAR-Generation/dataset/stage1_prepared/manifest.jsonl"
c = Counter()
with open(path, encoding="utf-8") as f:
    for line in f:
        if line.strip():
            row = json.loads(line)
            c[(row.get("dataset", "unknown"), row.get("split", "train"))] += 1
for key, value in sorted(c.items()):
    print(key, value)
PY
```

生成 32 对过拟合 manifest：

```bash
python tools/make_debug_manifest.py \
  --input "$FULL_MANIFEST" \
  --output data/debug_overfit_32/manifest.jsonl \
  --train-samples 32 \
  --val-samples 4 \
  --seed 42
```

生成单数据集 2k manifest。把 `SAR2Opt` 换成上面统计里样本数足够、且你最关心的数据集：

```bash
python tools/make_debug_manifest.py \
  --input "$FULL_MANIFEST" \
  --output data/debug_single_domain2k/manifest.jsonl \
  --dataset SAR2Opt \
  --train-samples 2048 \
  --val-samples 64 \
  --test-samples 64 \
  --seed 42
```

## 2. 版本 A：32 对原始 diffusion 过拟合

目的：验证当前原始 diffusion loss 能不能记住极小训练集。

```bash
CUDA_VISIBLE_DEVICES=0 python train_stage1.py \
  --config configs/debug_overfit32_base.yaml
```

重点看：

```text
runs/debug_overfit32_base/samples/epoch_00xx/*_panel.png
runs/debug_overfit32_base/metrics.jsonl
```

如果 32 对、几十到 200 epoch 都不能把结构记住，说明原始训练目标/条件注入不适合作为当前任务的快速收敛方案。

## 3. 版本 B：32 对 paired 辅助损失过拟合

目的：在 diffusion loss 外增加低噪声 x0 latent loss 和少量 image L1，测试模型结构是否具备拟合 OPT-SAR 配对关系的能力。

```bash
CUDA_VISIBLE_DEVICES=0 python train_stage1.py \
  --config configs/debug_overfit32_paired.yaml
```

重点看：

```text
x0
image
val_diffusion
samples/epoch_00xx/*_panel.png
```

判断：

- 如果版本 A 失败、版本 B 成功：问题主要是训练目标太弱，需要把 paired reconstruction 辅助目标纳入正式方案。
- 如果版本 B 也失败：优先怀疑 OPT 条件编码和 cross-attention 注入方式太弱，模型没有真正使用 OPT。

## 4. 版本 C：单数据集 2k paired 验证

目的：如果 32 对能拟合，继续验证单一数据域能不能稳定学习。这个实验排除多数据集异质性。

```bash
CUDA_VISIBLE_DEVICES=0 python train_stage1.py \
  --config configs/debug_single_domain2k_paired.yaml
```

如果单数据集 2k 明显好于全量混合数据，说明下一步应该按数据源分阶段训练，而不是直接混合 66 万对训练。

## 5. 面板含义

`*_panel.png` 从左到右为：

```text
OPT 输入 | 真实 SAR | 生成 SAR | 绝对差异图
```

生成 SAR 应首先学到真实 SAR 的大结构和亮散射区域，再谈 speckle 细节。若只出现随机 speckle 而不跟 OPT 结构对应，说明条件控制失败。
