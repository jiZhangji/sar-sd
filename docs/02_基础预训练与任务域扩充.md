# OPT-to-SAR：基础预训练与任务域扩充路线

## 1. 总体思路

将 OPT-to-SAR 生成拆为“基础预训练”和“任务域扩充”两个层次，并分三阶段实施：

1. **通用基础预训练**：使用大规模配对 OPT-SAR 数据训练通用的 OPT→SAR 潜空间扩散模型，不引入 layout，也不直接针对 M4 检测任务。
2. **M4 域适配**：将基础模型迁移到 M4 数据集，在无 layout 条件下微调，生成符合 M4 数据分布的 **SAR 图像 1**。
3. **layout 可控扩充**：以 SAR 图像 1 和 layout 为条件生成 **SAR 图像 2**，用于扩充目标检测训练数据。

这种递进方式先建立稳定的跨模态生成能力，再学习任务域风格和空间控制，避免 layout 条件过早干扰 OPT 与 SAR 的对应关系学习。

## 2. 第一阶段：大规模 OPT-to-SAR 扩散预训练

### 2.1 目标与边界

第一阶段只学习通用的光学到 SAR 跨模态映射：输入同一区域的光学图像，输出空间对应的 SAR 图像。该阶段不使用目标类别、检测标注或 layout。

### 2.2 数据

- 核心数据为严格配准、同名配对的大规模 OPT-SAR 图像。
- OPT 保留 RGB 信息，SAR 以单通道强度图读取。
- 几何增强必须对 OPT 和 SAR 同步执行，禁止破坏像素级空间对应。
- 数据划分应按地理区域隔离，避免相邻切片跨训练集与验证集造成泄漏。

目录约定：

```text
data_root/
├── train/
│   ├── opt/<sample_id>.png
│   └── sar/<sample_id>.png
└── val/
    ├── opt/<sample_id>.png
    └── sar/<sample_id>.png
```

### 2.3 模型与训练过程

采用标准 Latent Diffusion 训练范式：

1. 冻结的 VAE/Autoencoder 将真实 SAR 图像编码为潜变量 `z0`；后续可用 SAR 数据单独适配 VAE，使潜空间更适合灰度、散斑和强散射结构。
2. 随机采样时间步 `t` 和高斯噪声 `epsilon`，得到加噪潜变量 `zt`。
3. OPT 编码器将光学图像转换为 cross-attention 条件 token。
4. 条件 UNet 根据 `zt`、`t` 和 OPT 条件预测噪声。
5. 以标准扩散噪声预测 MSE 为主损失。

### 2.4 损失函数

总损失保持克制：

```text
L = lambda_diffusion * L_noise + lambda_physical * L_physical
L_physical = w_mean * L_local_mean
             + w_var * L_local_variance
             + w_cv * L_local_cv
```

- `L_noise`：预测噪声与真实噪声的均方误差。
- `L_local_mean`：约束局部后向散射强度水平。
- `L_local_variance`：约束局部强度起伏与散斑能量。
- `L_local_cv`：约束局部变异系数 `std / (mean + eps)`，描述相对散斑强度。

物理项从预测的干净 latent 解码到图像域后计算，并保留 UNet→VAE decoder→损失的梯度链路；VAE 参数本身保持冻结。为控制开销，可每隔若干训练步启用一次物理项。

## 3. 阶段间产物

第一阶段输出通用 OPT→SAR 权重和训练配置；第二阶段从该权重初始化，在 M4 配对数据上无 layout 微调；第三阶段再引入 `SAR 图像 1 + layout` 条件，完成可控细化和数据扩充。各阶段独立保存 checkpoint，便于消融比较基础能力、域适配收益和 layout 控制收益。

## 4. 第一阶段代码入口

```bash
cd layout_controlled_opt2sar_ldm
python train_stage1.py --config configs/stage1_opt2sar_pretrain.yaml
```

当前框架默认冻结 VAE，训练 OPT 条件适配器、控制残差和 UNet（可配置为冻结或 LoRA）。正式训练前需在配置中填写大规模配对数据路径及本地预训练 VAE/UNet 权重路径。
