# 第一阶段当前进展与训练运行存档

更新时间：2026-07-01

## 1. 三阶段任务定义

1. 第一阶段：在大规模配对OPT-SAR数据上预训练通用OPT→SAR潜在扩散模型，不引入layout和M4任务条件。
2. 第二阶段：迁移到M4数据集，无layout微调并生成SAR图像1。
3. 第三阶段：输入SAR图像1和layout，生成用于数据扩充的SAR图像2。

当前仓库只实现第一阶段。M4-SAR被明确排除在第一阶段manifest之外。

## 3. 模型结构

- 使用Stable Diffusion风格的Latent Diffusion训练方式。
- 冻结 `stabilityai/sd-vae-ft-mse` VAE，将单通道SAR复制为三通道后编码到latent。
- 使用SD 1.5预训练条件UNet。
- OPT经过轻量CNN编码为8×8 cross-attention tokens。
- 数据集、传感器、极化、GSD和入射角编码为metadata token。
- UNet预测SAR latent中的DDPM噪声。
- 主损失是标准扩散噪声MSE。
- 辅助物理项只包含局部均值、局部方差和局部变异系数。

## 4. 物理损失策略

物理损失不会在最开始启用：

```yaml
physical_warmup_steps: 5000
physical_interval: 4
physical_max_timestep: 500
adaptive_physical: true
physical_target_grad_ratio: 0.05
physical_lambda_max: 0.10
physical_max_samples_per_rank: 8
```

原因是训练初期预测SAR不稳定，过早加入统计监督会干扰扩散目标。第5000步后，仅在中低噪声样本上每4步计算一次。

物理分支每卡最多抽8个有效样本执行带梯度VAE解码；当前batch内其他样本仍全部参与扩散主损失。该限制只控制辅助分支峰值显存。

自适应λ依据UNet噪声预测输出处的两项梯度比更新，并在DDP rank间同步。

## 5. 双H100最终运行配置

硬件：2×H100 80GB。

经实测确定：

```text
每卡batch size：48
全局batch size：96
物理分支每卡样本上限：8
物理显存测试峰值：约68GB/卡
训练epoch：20
混合精度：BF16
```

约660172个训练样本对应：

```text
每轮优化步数约：660172 / 96 ≈ 6877
20轮约：137540步
```

训练前使用8步显存测试，物理损失从第0步、每步开启。BS=32且不限制物理样本时曾达到约77GB并OOM；限制物理样本后，BS=48、每卡8个物理样本峰值约68GB。

## 8. 断点续训

`--epochs`表示总目标轮数，不是额外轮数。例如从第10轮继续到第20轮：

```bash
BATCH_SIZE=48 EPOCHS=20 nohup bash train_h100_2gpu.sh \
  --resume-from runs/stage1_opt2sar_pretrain/checkpoints/last.pt \
  > runs/stage1_opt2sar_pretrain/train.log 2>&1 &
```

当前没有学习率调度器，默认固定学习率为 `1e-5`。如果接近收敛后续训，可显式加入：

```text
--lr 5e-6
```

显式传入的学习率会在恢复optimizer状态后覆盖checkpoint中的旧学习率。

## 10. 当前仍需关注

- 自然图像SD VAE可能损失SAR散斑、强散射点和小目标细节，需要单独评估SAR重建质量。
- 当前OPT条件编码器是轻量单尺度CNN，精细空间对齐能力仍需验证。
- 当前图像统一resize为512×512，GSD token不能完全替代物理尺度一致的裁剪与重采样。
- 需要根据第2、4、6轮固定验证样本判断生成质量，而不能只看扩散loss。
- 如果第10轮以后仍明显改善，可训练到20轮；若验证结果已稳定，可提前停止。
