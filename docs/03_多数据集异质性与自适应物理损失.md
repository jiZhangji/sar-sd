# Stage 1 多数据集异质性与自适应物理损失

## 1. 目标

Stage 1 混合多个配对 OPT-SAR 数据集时，不把所有样本简单 resize 后视为同一分布。框架显式记录数据集、OPT/SAR 传感器、GSD、极化和入射角，并确保标准扩散噪声预测始终是主任务。

## 2. 自适应物理项

总损失只有两项：

```text
L = L_diffusion + lambda_phy * L_physical
```

`L_physical` 内部只包含局部均值、局部方差和局部变异系数。控制器定期测量两项损失在 UNet 输出层产生的梯度范数：

```text
lambda_raw = target_ratio * ||grad(L_diffusion)|| / (||grad(L_physical)|| + eps)
lambda_phy = clip(EMA(lambda_raw), lambda_min, lambda_max)
```

默认目标比例为5%，上限为0.1。训练初期只使用扩散损失；物理项只在中低噪声时间步计算，避免从高噪声 latent 反推 `z0` 时产生不可靠监督。控制器状态随 checkpoint 保存和恢复。

推荐消融：仅扩散损失、固定权重、受限自适应权重；每组再比较 `[mean, variance]` 与 `[mean, variance, cv]`。

## 3. 分辨率和物理窗口

manifest 为每个样本保存 `opt_gsd` 和 `sar_gsd`。局部统计窗口不再固定为9像素，而是由目标地面范围换算：

```text
window_px = odd_clip(round(window_meters / sar_gsd), min_px, max_px)
```

因此不同分辨率样本的统计约束尽量对应相近的真实地面尺度。正式数据预处理仍应优先按地面覆盖范围裁剪和配准，GSD条件不能修复错误配准。

## 4. 元数据条件

OPT条件由图像token和一个元数据token组成。元数据token编码：

- 数据集ID（带dropout，仅作弱辅助）；
- OPT和SAR传感器；
- 极化方式；
- `log(GSD)` Fourier特征；
- 入射角及元数据是否已知。

物理字段是主要条件。训练时随机将部分数据集ID替换为 `unknown`，降低模型死记数据集风格的风险。

## 5. 数据平衡

若数据集 `d` 有 `N_d` 个样本，采样概率为：

```text
p(d) proportional to N_d ** temperature
```

- `temperature=1`：完全按样本数量；
- `temperature=0`：各数据集等概率；
- 默认 `0.5`：兼顾百万级数据与小规模高分辨率数据。

## 6. 辐射处理边界

当前冒烟数据是展示用8位图像，manifest 标记为 `display_uint8`。正式训练应尽可能从原始产品统一到 `sigma0` 或 `gamma0` 的dB表达，并使用训练集确定的传感器级固定裁剪范围；不能对每张图独立Min-Max，否则局部均值等物理统计失去可比性。

## 7. 下载支持状态

- SEN1-2：支持从TUM官方FTP按数量下载。
- SAR2Opt：支持从公开Hugging Face镜像按数量下载。
- WHU-OPT-SAR和SAR-1M：支持从Hugging Face下载整包，并要求显式 `--full`。
- SOMA-1M test：支持作者官方Google Drive文件。
- 3MOS与QXS-SAROPT：官方入口需要网盘或申请，脚本只给出官方入口，不伪造直链。

没有把论文拼图或网页预览图当作原始训练样本。
